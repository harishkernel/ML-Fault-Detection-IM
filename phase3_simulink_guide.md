# Phase 3 — Simulink Digital Twin Assembly Guide

> **Goal**: Wire a complete real-time fault-detection loop on the Simulink canvas that streams vibration data → buffers it → runs the MATLAB Function classifier → displays the result.

---

## Block Inventory (Library Browser search names)

| # | Block Name (Library Path) | Purpose |
|---|---------------------------|---------|
| 1 | **From Workspace** — `Simulink > Sources > From Workspace` | Streams the vibration time-series into the model |
| 2 | **Buffer** — `DSP System Toolbox > Signal Management > Buffers > Buffer` | Accumulates streaming samples into a fixed-size frame |
| 3 | **MATLAB Function** — `Simulink > User-Defined Functions > MATLAB Function` | Runs the Phase 2 fault-detection code |
| 4 | **Display** — `Simulink > Sinks > Display` | Shows the current `faultFlag` value (0 or 1) |
| 5 | **Scope** — `Simulink > Sinks > Scope` | Plots the fault flag over time |
| 6 | **Lamp** *(optional, Dashboard)* — `Simulink > Dashboard > Lamp` | Visual RED/GREEN fault indicator |

## Step-by-Step Wiring

### Step 1 — Prepare Workspace Data

Before running the model, execute this in the MATLAB Command Window:

```matlab
% Create a Simulink-compatible timeseries of faulty data for testing
Fs = 12000; duration = 2; t = (0:1/Fs:duration-1/Fs)';
N_balls = 9; fr = 1750/60; d_ball = 0.311; D_pitch = 1.5; alpha = 0;
f_BPFO = (N_balls/2)*fr*(1 - (d_ball/D_pitch)*cos(alpha));
vibData = 0.5*randn(size(t)) + 1.5*sin(2*pi*f_BPFO*t);
simInput = timeseries(vibData, t);
```

### Step 2 — Place and Configure "From Workspace"

1. Drag **From Workspace** onto the canvas.
2. Double-click → set parameters:
   - **Data**: `simInput`
   - **Sample Time**: `1/12000`  (= 8.3333e-05)
   - **Interpolate data**: ☑ checked
   - **Output after final data**: `Extrapolation`
3. **Output type** will be *scalar* (one sample per time step).

### Step 3 — Place and Configure "Buffer"

1. Drag **Buffer** onto the canvas.
2. Double-click → set parameters:
   - **Output buffer size (per channel)**: `12000`
   - **Buffer overlap**: `0`
   - **Initial conditions**: `0`
3. The Buffer collects 12 000 individual samples (1 full second at 12 kHz) and outputs them as a `[12000 × 1]` column vector.

### Step 4 — Place and Configure "MATLAB Function"

1. Drag **MATLAB Function** onto the canvas.
2. Double-click to open the MATLAB Function Editor.
3. **Delete** the auto-generated stub and **paste the entire contents** of [`phase2_simulink_function.m`](file:///c:/Users/haris/OneDrive/Documents/EEC/Year_3/Sem_6/Innovation%20Lab/phase2_simulink_function.m).
4. Save and close the editor.
5. Verify the block now shows:
   - **Input port**: `vibrationBuffer`
   - **Output port**: `faultFlag`

### Step 5 — Place Sinks

1. **Display** block — drag onto canvas (shows numeric 0 or 1).
2. **Scope** block — drag onto canvas (plots fault flag over simulation time).
3. *(Optional)* **Lamp** block — drag onto canvas for a dashboard indicator.

### Step 6 — Wire the Connections

```
From Workspace  ──►  Buffer  ──►  MATLAB Function  ──┬──►  Display
                                                      ├──►  Scope
                                                      └──►  Lamp (optional)
```

| Source Block | Source Port | → | Destination Block | Destination Port |
|---|---|---|---|---|
| From Workspace | Output (1) | → | Buffer | Input (1) |
| Buffer | Output (1) | → | MATLAB Function | `vibrationBuffer` |
| MATLAB Function | `faultFlag` | → | Display | Input (1) |
| MATLAB Function | `faultFlag` | → | Scope | Input (1) |
| MATLAB Function | `faultFlag` | → | Lamp | Input (1) |

### Step 7 — Model Configuration

Open **Modeling > Model Settings** (Ctrl+E) and set:

| Parameter | Value | Reason |
|---|---|---|
| **Solver > Type** | `Fixed-step` | Required for code generation and real-time consistency |
| **Solver > Solver** | `discrete (no continuous states)` | All signals are discrete |
| **Solver > Fixed-step size** | `1/12000` | Matches your 12 kHz sample rate |
| **Simulation > Stop time** | `2` | 2 seconds of test data |

### Step 8 — Ensure Files Are on MATLAB Path

The following files **must** be in the Current Folder (or on the MATLAB path) before running:

- `mySVM.mat` — the trained SVM model (generated by Phase 1)
- `phase2_simulink_function.m` — referenced indirectly by the MATLAB Function block

### Step 9 — Run the Simulation

1. Click the green **Run** button (▶) or press `Ctrl+T`.
2. Observe:
   - **Display** block should show `1` (fault detected) for the synthetic faulty signal.
   - **Scope** should show a flat line at `1`.
   - **Lamp** (if used) should turn to the fault-state colour.

## Troubleshooting

| Issue | Likely Cause | Fix |
|---|---|---|
| `"Undefined function 'loadLearnerForCoder'"` | Missing Statistics & ML Toolbox | Install via Add-On Explorer |
| `"Unable to resolve coder.load"` | `mySVM.mat` not on path | `cd` to the project folder or `addpath(...)` |
| Buffer output size mismatch | Buffer size ≠ 12000 | Re-check Buffer block parameters |
| Simulation extremely slow | Solver step too small | Confirm Fixed-step = `1/12000`, not smaller |
| Lamp not responding | Lamp not bound to signal | Right-click Lamp → **Connect to Signal** → select `faultFlag` |
